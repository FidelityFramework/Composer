# I-01: Socket I/O Basics

> **Sample**: `23_SocketBasics` | **Status**: Planned | **Depends On**: A-01-19 (Async), A-04-22 (Regions)

## 1. Executive Summary

This PRD introduces socket I/O — the foundation for network programming. Sockets enable TCP/UDP communication using POSIX libc functions, accessed via Farscape-generated `Fidelity.Libc` bindings.

**Key Insight**: Sockets are file descriptors. The same `read`/`write` functions from `Fidelity.Libc.IO` work for sockets as for files. This PRD uses additional libc functions: `socket`, `bind`, `listen`, `accept`, `connect`.

**Architecture**: All socket operations use the **ExternCall pathway** — they are calls to libc functions via `[<FidelityExtern("c", "socket")>]` declarations generated by Farscape. No CCS intrinsics, no hardcoded syscall numbers.

## 2. Language Feature Specification

### 2.1 Socket Creation

```clef
open Fidelity.Libc.Socket.Api

let sock = socket AF_INET SOCK_STREAM 0
```

Creates a TCP socket. Returns `Result<int, CError>` via L2 errno-wrapping.

### 2.2 Server Pattern

```clef
open Fidelity.Libc.Socket.Api
open Fidelity.Libc.IO.Api

let server () =
    match socket AF_INET SOCK_STREAM 0 with
    | Ok sock ->
        bind sock (SockAddr.ipv4 "0.0.0.0" 8080) |> ignore
        listen sock 10 |> ignore

        match accept sock with
        | Ok client ->
            let buffer = Region.alloc<byte> region 1024
            let n = read client buffer 1024
            // ... process ...
            close client |> ignore
        | Error e ->
            Console.writeln "Accept failed"

        close sock |> ignore
    | Error e ->
        Console.writeln "Socket creation failed"
```

### 2.3 Client Pattern

```clef
open Fidelity.Libc.Socket.Api
open Fidelity.Libc.IO.Api

let client () =
    match socket AF_INET SOCK_STREAM 0 with
    | Ok sock ->
        connect sock (SockAddr.ipv4 "127.0.0.1" 8080) |> ignore
        write sock data len |> ignore
        close sock |> ignore
    | Error e ->
        Console.writeln "Connection failed"
```

## 3. Binding Generation (Farscape)

### 3.1 Socket Functions in libc.pilot.toml

Socket functions are part of the libc pilot TOML, in a dedicated namespace:

```toml
[[namespace]]
name = "Fidelity.Libc.Socket"
description = "Socket creation, binding, and connection"
library = "c"
functions = ["socket", "bind", "listen", "accept", "connect", "shutdown",
             "setsockopt", "getsockopt", "getaddrinfo", "freeaddrinfo",
             "inet_pton", "inet_ntop", "htons", "ntohs", "htonl", "ntohl"]

[error_conventions]
default = "errno"
```

### 3.3 Generated L2 Wrappers (SocketApi.clef)

```clef
module Fidelity.Libc.Socket.Api

open Fidelity.Libc.Socket
open Fidelity.Libc.Errno

/// Create a socket. Returns Result with file descriptor or CError.
let socket (domain: int) (``type``: int) (protocol: int) : Result<int, CError> =
    let fd = Fidelity.Libc.Socket.socket domain ``type`` protocol
    if fd < 0 then Error (captureError ())
    else Ok fd

/// Bind a socket to an address.
let bind (sockfd: int) (addr: nativeint) (addrlen: int) : Result<unit, CError> =
    let result = Fidelity.Libc.Socket.bind sockfd addr addrlen
    if result < 0 then Error (captureError ())
    else Ok ()
```

### 3.4 Socket Constants

Socket constants (`AF_INET`, `SOCK_STREAM`, etc.) are C macros, discovered by Farscape's macro extraction and emitted as module-level constants:

```clef
module Fidelity.Libc.Socket.Constants

let AF_INET = 2
let AF_INET6 = 10
let SOCK_STREAM = 1
let SOCK_DGRAM = 2
```

### 3.5 SockAddr Construction

`SockAddr` construction is a **Layer 3 concern** — it requires both `Fidelity.Libc` (for `inet_pton`, `htons`, `malloc`) and struct layout knowledge. This belongs in a composition library:

```clef
module Fidelity.Libc.Net

open Fidelity.Libc.Socket
open Fidelity.Libc.Memory

/// Construct a sockaddr_in for IPv4.
/// Allocates from the region and returns pointer to populated struct.
let sockAddrIpv4 (ip: nativeint) (port: int) : nativeint =
    let addr = malloc 16  // sizeof(sockaddr_in)
    // ... populate sin_family, sin_port (htons), sin_addr (inet_pton) ...
    addr
```

## 4. Compiler Pipeline

No CCS or Composer modifications needed. Socket functions use the same ExternCall pathway as all other Farscape-generated bindings (D-01).

The pipeline for `socket AF_INET SOCK_STREAM 0`:
1. CCS resolves `socket` to `Fidelity.Libc.Socket.socket` via `open` — recognizes `[<FidelityExtern>]`
2. Baker: no special handling
3. `PlatformBindingResolution`: metadata → `ExternCall ("c", "socket")`
4. Alex: `pExternCallResolved` → `func.call @socket(...)` with `"link" = "c"` attribute

### 6.1 Sample Code

```clef
module SocketBasicsSample

open Fidelity.Libc.Socket.Api
open Fidelity.Libc.Socket.Constants

let main _ =

    match socket AF_INET SOCK_STREAM 0 with
    | Ok sock ->

        match bind sock (SockAddr.ipv4 "0.0.0.0" 8080) with
        | Ok () ->

            match listen sock 10 with

                Console.writeln "Listening..."
                Console.writeln "Server setup complete"
            | Error e ->
                Console.writeln "Listen failed"

        | Error e ->
            Console.writeln "Failed to bind"

        close sock |> ignore
        0

        Console.writeln "Failed to create socket"
        1
```

## 7. Error Handling

Socket operations return -1 on error with errno set. The L2 wrappers from `Fidelity.Libc.Socket.Api` automatically capture errno and return `Result<T, CError>`, using the same errno infrastructure as all other libc wrappers (Farscape Phase 1.3).
